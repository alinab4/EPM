# Employee Performance Management System

## Project Overview
This project is an Employee Performance Management (EPM) system designed to streamline and automate the process of performance reviews, feedback, and KPI tracking within an organization. It provides a platform for employees, managers, and administrators to interact and manage performance-related activities.

## Features
- **User Authentication:** Secure user registration and login.
- **Role-Based Access Control:** Different views and permissions for Employees, Managers, and Admins.
- **Performance Reviews:** Create, view, and manage performance reviews.
- **360-Degree Feedback:** Allow employees to give and receive feedback.
- **KPI Tracking:** Define and track Key Performance Indicators (KPIs).
- **Admin Dashboard:** A centralized dashboard for administrators to manage users and system settings.

## Getting Started

### Prerequisites
- Python 3.8+
- PostgreSQL

### Installation
1.  **Clone the repository:**
    ```bash
    git clone <repository-url>
    cd <repository-directory>
    ```
2.  **Create a virtual environment:**
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
    ```
3.  **Install the dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

### Database Setup
1.  **Create a PostgreSQL database.** The default database name is `epm_db`, but you can choose a different name.

### Database Configuration
- The application is configured to connect to a PostgreSQL database. You will need to make sure you have a running PostgreSQL instance.
- To connect to your database, you can set the following environment variables. The easiest way to manage these is to create a `.env` file in the root of the project:

-  `DB_USER`: Your PostgreSQL username (default: `postgres`)
-  `DB_PASSWORD`: Your PostgreSQL password (default: `postgres`)
-  `DB_HOST`: The database host (default: `localhost`)
-  `DB_PORT`: The database port (default: `5432`)
-  `DB_NAME`: The name of your database (default: `epm_db`)

If you want to use a different database name, simply change the `DB_NAME` environment variable.
3.  **Initialize the database:**
    - Run the `setup_backend.py` script to create the database tables:
    ```bash
    python setup_backend.py
    ```

### Running the Application
- To start the application, run the following command in your terminal:
    ```bash
    uvicorn app.main:app --reload
    ```
- The application will be available at `http://127.0.0.1:8000`.

## API Endpoints
The application provides a RESTful API for managing users, performance reviews, feedback, and KPIs. The API documentation is automatically generated by FastAPI and is available at `http://127.0.0.1:8000/docs`.

## Functionality
- **User Management:** The `app/routes/users.py` file contains the API endpoints for creating, retrieving, updating, and deleting users.
- **Authentication:** The `app/routes/auth.py` file handles user authentication and token generation.
- **Performance Reviews:** The performance review functionality is implemented in the `app/routes/performance.py` file.
- **Feedback:** The feedback-related API endpoints are located in the `app/routes/feedback.py` file.
- **KPIs:** The `app/routes/kpi.py` file contains the API endpoints for managing KPIs and KPI results.

## User Registration
- **Employee and Manager Self-Registration:** Employees and managers can register themselves using the `/api/auth/register` endpoint. By default, new users are assigned the "Employee" role.
- **Admin User Creation:** Administrators can manually create new users, including other administrators and managers, via the `/api/users/` endpoint. This gives administrators full control over user management.

### 2. API Endpoints
### User Management
- `GET /api/users/` - List all users (Admin only)
- `POST /api/users/` - Create user
- `PUT /api/users/{id}` - Update user (Admin only)
- `DELETE /api/users/{id}` - Delete user (Admin only)
- `GET /api/users/dashboard/admin` - Admin dashboard
- `GET /api/users/dashboard/manager` - Manager dashboard
- `GET /api/users/dashboard/employee` - Employee dashboard
### Authentication
- `POST /api/auth/login` - Login user
- `POST /api/auth/register` - Register new user (Admin only)
### Performance Reviews
- `POST /api/performance/reviews/` - Create performance review
- `GET /api/performance/reviews/` - List all performance reviews
- `GET /api/performance/reviews/{id}` - Get performance review by ID
- `PUT /api/performance/reviews/{id}` - Update performance review
- `DELETE /api/performance/reviews/{id}` - Delete performance review
### Feedback
- `POST /api/feedback/` - Post feedback
- `GET /api/feedback/` - List all feedback
- `GET /api/feedback/{id}` - Get feedback by ID
- `PUT /api/feedback/{id}` - Update feedback
- `DELETE /api/feedback/{id}` - Delete feedback
### KPI Management
- `POST /api/kpis/` - Create KPI
- `GET /api/kpis/` - List all KPIs
- `GET /api/kpis/{id}` - Get KPI by ID
- `PUT /api/kpis/{id}` - Update KPI
